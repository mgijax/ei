MGI EDITING INTERFACE: Design and Requirements - Editing Commands


Introduction

For edit forms with multiple forms, the scope of the Control buttons are always within the current form:
  • Reference/Primary Markers and Experiments
  • Molecular Segments and Molecular Segment References

Search

 The Search function:
  • Constructs an SQL query based on values entered into the edit form fields.
  • Executes the SQL query.
  • Returns a list of unique, sorted results into the Search Results list.
  • Selects the first item in the Query Results. This item is now ready for modification.

The Search function can be initiated in several ways:

  • clicking on the Search button in the upper right-hand corner of the edit form (the Control area)
  • selecting the Search option from the Commands pulldown menu at the top of the edit form
  • hitting Control-s

Notes

  • When entering values into Tables for searching, always enter the values into the first row of the Table. Only the first row of the Table is scanned for search criteria during the construction of the query.

    Refer to Table Processing for more information about Tables.

  • A Search in progress can be cancelled by clicking on the Cancel button in the Search dialog. A search can be cancelled any time during the display of this dialog.

  • Not all edit form fields have Search capabilty. Refer to the individual requirements document for each edit form for a list of exceptions.

  • Use the Search All option in an Option Menu to specify a wildcard or "match any" value.

  • All SQL search options are valid (% = wildcard, [] = range, etc.)

  • All SQL errors will be returned to the user in a Status message dialog.

Add

 The Add function:
  • Inserts a new record into the database for the given object.

  • If the add is successful, then:
    • If the object is accessionable, an MGI Accession number is generated by the system.
    • If the object is a Reference, then a J: Accession number is either generated by the system or assigned by the user.
    • The new record is inserted into the Search Results list.
    • Finally, either:
      • The edit form is cleared in preparation for the next event.
      • The edit form context is switched to the detail form so that detail records can be added. This applies to:
        • Molecular Segments
        • Mapping

  • If the add is unsuccessful, then an error message is displayed explaining why.

The Add function can be initiated in several ways:

  • clicking on the Add button in the upper right-hand corner of the edit form (the Control area)
  • selecting the Add option from the Commands pulldown menu at the top of the edit form
  • hitting Control-a

Notes

  • If a record is currently selected, then the Add is denied.

  • If required fields are empty, then the Add is denied.

  • Defaults are invoked for any fields which do not have values entered.

  • All SQL errors will be returned to the user in a Status message dialog.

Modify

 The Modify function:
  • Modifies the database record for the given object.

  • If the modification is successful, then the object is re-selected and the recent modifications are reflected in the current object view.

  • If the modification is unsuccessful, then an error message is displayed explaining why.

The Modify function can be initiated in several ways:

  • clicking on the Modify button in the upper right-hand corner of the edit form (the Control area)
  • selecting the Modify option from the Commands pulldown menu at the top of the edit form
  • hitting Control-m

Notes

  • If a record is not currently selected, then the Modify is denied.

  • If required fields are empty, then the Modify is denied.

  • All SQL errors will be returned to the user in a Status message dialog.

Delete

 The Delete function:
  • Verifies that the user really wants to perform the Delete (a last chance to say no).

  • Deletes the database record for the given object.

  • If the deletion is successful, then the object is removed from the Search Results list and the edit form is cleared.

  • If the delete is unsuccessful, then an error message is displayed explaining why.

The Delete function can be initiated in several ways:

  • clicking on the Delete button in the upper right-hand corner of the edit form (the Control area)
  • selecting the Delete option from the Commands pulldown menu at the top of the edit form
  • Hitting Control-d

Notes

  • If a record is not currently selected, then the Delete is denied.

  • All SQL errors will be returned to the user in a Status message dialog.
