GXD EDITING INTERFACE: Design and Requirements - Images


Introduction

(This information has been extracted from the GXD Schema document)

An image is associated with a reference and figure number within the reference, describes its format and (x,y) dimensions. An image may have a caption and/or a legend.

In a publication, images are usually grouped into panels, where each panel is one figure. Individual images within a panel are referred to as "panes". Panes are usually labelled A, B, C, etc. One scanning operation captures the entire panel. One figure (panel) in a reference equals one image in the database.

Actual images (pixels) are not stored in MGI but are stored in the Pixel Image Database and referenced by a Pixel Image Accession number.

Editing Form

GIF Image

Accession Numbers

  1. MGI:, system-generated
  2. MGI Image, cross reference to Pixel Image Database
    Additional constraints:
    • MGI Image accession numbers can only be added via the Pixel Image Database entry form.
    • MGI Image accession numbers can be deleted but not modified via the GEI.
    • One and only one MGI Image accession number can be assigned per GXD Image record.

Data Verifications

  1. J Numbers (see Data Verifications section on J Numbers)

Fields

  • Required
    1. J Number
    2. Figure Label
    3. Pane Label Field Type

  • Not Required
    1. X Dimension
    2. Y Dimension
    3. Note or Legend
    4. Copyright Note
    5. Pane Label

Defaults

None

Deletions

An Image cannot be deleted if it is referenced by one of the following objects:
  1. Assay

Miscellaneous

  1. An Image record must exist even if the actual image is never scanned into the database.

  2. When an MGI Image accession number is deleted, the X Dimension and Y Dimension fields are nullified.
