MGD EDITING INTERFACE: Design and Requirements - References


Introduction

References, also known as the Master Bibliography, constitute one of the backbones of the MGD database. No data is input into MGD without a supported Reference. Within the data entry process, the Reference is the first bit of information entered into the database.

Along with the MGI accession number, an internal accession number (J Number) is also generated. This J Number facilitates the filing and tracking of the References within the Data Processing group.

The Master Bibliography is updated from one external source, NLM (The National Library of Medicine). Data loads from each of this resources are automatically processed and loaded into the Master Bibliography at regular intervals.

Data Verifications

  1. Data duplications are checked after the user executes an add or modification:
    1. A J Number will be checked for duplication against the Reference table.
    2. A Reference will be checked for duplication against the Reference table using the following criteria:
      • Journal
      • Year
      • Volume
      • First Page
    3. Modifications denied on invalid entries.

  2. If a Journal cannot be found in the database, then a list of potential options is presented to the user. A match is made using the first 3 characters of the Journal entered. (Journals are currently not normalized into a controlled vocabulary list).

  3. J: Accession numbers are checked after the user executes an add or modification. If the J: is a duplicate, the add/modification is denied.

  4. Dates (see Data Verifications section on Dates)
    • The Reference Date field must contain a valid Year in the format YYYY. Modifications denied on invalid entries.

  5. The Primary Author is derived from the Authors field after the user TABS out of the Authors field. A semicolon + space separates each Author in the Authors field if there is more than one Author (ex. Smith JE; Jones BC)

  • Required

    2. In NLM?
    3. Review Status
    4. Date

  • Not Required
    1. Authors
    2. Title
    3. Journal

    8. Notes

    12. Place
    13. Publisher
    14. Series/Edition

Defaults

  1. Reference Type default is Article

  2. In NLM? default is Yes, except:
    • If Journal is Mouse Genome or Mouse News Lett then default is Never

  3. Review Status default is Peer Reviewed, except:
    • If Title contains the term abstr, then default is Unreviewed
    • If Page contains the term abstr, then default is Unreviewed
    • If Title contains the term Personal Comm, then default is Unreviewed
    • If Journal contains the term Personal Comm, then default is Unreviewed
    • If Journal contains the term Database Release, then default is Unreviewed
    • If Journal is Mouse News Lett, then default is Unreviewed

Deletions

A Reference cannot be deleted if it is referenced by any of the following objects:
  1. Marker
  2. Experiment
  3. Molecular Probe & Segment
  4. Homology
  5. MLC entry
  6. Expression Assay
  7. Allele

Reference Data Sets

Each Reference can be assigned to one or more data sets in MGD:
  1. Probes (Molecular Probes & Segments)
  2. Mapping
  3. MLC/Alleles (Phenotype)
  4. Homology
  5. Expression (GXD)
  6. Tumor
  7. SCC (Strain Characteristics Catalogue)
  8. Matrix (Biochemical Alleles)
  9. Chromosome Committee
  10. Nomenclature

For data sets 1-5, one of the following statuses is determined automatically by the system:

  1. Used
  2. Not Used

The user may also elect to status data sets 1-5 as Never Used.

The only status option for data sets 5-10 is Never Used since there is no way for the system to determine a Used/Not Used status for these data sets. (However, if any of these data sets were incorporated into MGD, then this behavior could be changed to automatically determine their status.)

To search for References by data set:

  1. Choose the data set(s)
  2. Choose the operator
    • Equals will search for References where the selected data set is the only data set assigned to the Reference. Only select one data set when using the Equals option.
    • Contains will search for References where any of the selected data sets are assigned to the Reference.

  1. If the Reference Type of the record is Book, then the following fields are relevant:

  2. The last assigned J: is always displayed under the Last J# label in the right-hand panel of the form.
