MGD EDITING INTERFACE: Design and Requirements - Molecular Probes & Segments


Introduction

Molecular Segments contains:
  • cDNA
  • genomic
  • DNA (construct)
  • mitochondrial
  • oligo
  • primer

All Molecular Segments which are not designated as primers have a Library record which defines:

  • Species
  • Strain
  • Tissue
  • Cell Line
  • Age
  • Sex
  • Tissue Treatment

Molecular Segments have relationships with Markers:

  • A, a Marker amplifies a primer
  • E, a Marker encodes a probe if it hybridizes 100% to the probe
  • H, a Marker hybridizes to a probe
  • M, designation for MIT primers
  • P, a putative association derived from dbEST GenBank IDs

Each Molecular Segment may have a list of References which may contain RFLP information, including nucleotide (GenBank) accession numbers.

Molecular Segments are loaded from external sources as well as manual input. These updates occur on a regular basis (quarterly, monthly, etc.):

  • MIT primers
  • I.M.A.G.E. clones

Defaults

  1. Species; default is mouse, laboratory
  2. Vector Type; default is Not Specified
  3. Age; default is Not Specified
  4. Sex; default is Not Specified
  5. If Tissue is unfertilized egg, then Age default is Not Applicable
  6. If Age is Not Applicable, then Tissue must be unfertilized egg

Deletions

There are no restrictions on deleting a Molecular Probe & Segment.

Library

A Library selection populates the Library text field and performs a lookup of:
  • Species
  • Strain
  • Tissue
  • Cell Line
  • Age
  • Sex
  • Tissue Treatment

To edit a non-anonymous Library, the user can also use the Molecular Probe Libraries form. To access this form, choose Edit->Molecular Probe Libraries.

Miscellaneous

  1. To view/edit Molecular Segment master records, set the References toggle OFF.

  2. To view/edit Molecular Segment References records, set the References toggle ON.

  3. The Marker/Probe relationship (R column in the Marker table):
    • must be H (hybridizes) for Probes of non-mouse source. This includes Not Specified and Not Applicable sources.
    • can only be P during an EST bulk load.
    • can be updated to another relationship from P

  4. The KFMemorial button places the following text in the Notes field: 
    This clone is derived from a gene that produces alternate transcripts.

  5. Strains in the RFLV table on the Reference form must be entered in the following format: 
     Strain1, Strain2, Strain3

    The delimiter "comma space" MUST be used to separate two distinct Strains. If the correct delimiter is not used, then the Strains will be treated as one Strain.

  6. Tabbing through the Endo and Marker columns in the RFLV table will cause the previous row's values to be copied to the current row for those columns.

  7. The deletion of a master Molecular Segment record will delete the Molecular Segment and all of its References.

  8. Editing of WashU clone records added from the electronic dbEST load is restricted. These electronically loaded records are distinguished by the reference J:57656. Users may add new Accession numbers to any WashU clone record, but they may not edit Accession numbers, clone name or source information for any WashU clone which is referenced by J:57656.

    • If the user attempts to edit the Accession numbers of a WashU clone record from the dbEST load, the following error message will appear: 
      	Accession ID is referenced by J:57656, WashU-HHMI Database Download.
	Cannot edit.
    • If the user attempts to edit the Source information of a WashU clone record from the dbEST load, the following error message will appear: 
      	Molecular Segment is referenced by J:57656, WashU-HHMI Database Download.
	Cannot modify the Source information.
    • If the user attempts to edit the name of a WashU clone record from the dbEST load, the following error message will appear: 
      	Molecular Segment is referenced by J:57656, WashU-HHMI Database Download.
	Cannot modify the name.
