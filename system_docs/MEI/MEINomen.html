MGD EDITING INTERFACE: Design and Requirements - Nomenclature


Introduction

The Nomen DB is used by the MGI informatics group to enter novel and reserved mouse gene symbols and to broadcast these symbols to MGD.

Refer to TR 1291 Requirements for more information.

Accession Numbers

  1. EC, cross reference to Enzyme Commission number at ExPASy
  2. Sequence, cross reference to GenBank, EMBL, DDBJ (nucleotide sequences)

Data Verifications

  1. Event (see Data Verifications section on Option Menus)
    • The event withdrawal is not valid during an add or modification. However, this value is used for historical data and entries exist which use this assignment value.

  2. Status (see Data Verifications section on Option Menus)
    • Pending; the symbol has not been approved by the Mouse Nomenclature Committee
    • Deleted; the symbol has been removed from the Nomenclature process
    • Reserved; the symbol has been reserved for the Author and is not to be publicized
    • Approved; the symbol has been approved by the Mouse Nomenclature Committee and is ready to broadcast to MGD
    • Broadcast; the symbol has been transferred to MGD

  3. J Numbers (see Data Verifications section on J Numbers)

  4. Markers
    • If the Marker symbol exists in MGD or Nomen, then a warning is displayed to the user. Modifications allowed.
    • If the Marker symbol exists in MGD with the "-pending" suffix, then a warning is displayed to the user. Modifications allowed.

  5. Marker Type (see Data Verifications section on Option Menus)

  6. Chromosome (see Data Verifications section on Option Menus)
    • Chromosome W and RE are not valid during an add or modification. However, these values are used for historical data and entries exist which use these assignment values.

  7. Submitted By and Broadcast By (see Data Verifications section on Option Menus)

    These option menus are dynamically created from the Nomenclature database sysusers table. To be added to the lists:

    • Contact John Gilbert (DBA) and request a Sybase login (if you do not already have one).
    • Contact Lori Corbani (SE) and ask to be included in the lists (this involves a modification to the Nomen User view and permissions on stored procedures).

    The retired_editors user has been assigned to all historical data migrated from the original FileMaker Pro database, since this database did not track the submitter information. The retired_editors user cannot be selected for new entries and is only used in reference to migrated data.

    Inactive logins (editors who have left the group) must still be available for searching and retrieving data, but are invalid for new entries.

  8. Gene Family (see Data Verifications section on Lookup Lists)

  9. Dates (see Data Verifications section on Dates)
    • Broadcast Date

  10. Review? (see Data Verifications section on Option Menus)

    Modifying this value will set the "Is Review Article?" flag in the MGD Master Bib record of the reference.

  11. Broadcast? (see Data Verifications section on Option Menus)

    If Yes, then the reference will be loaded into MGD during a broadcast. If No, then the reference will not be loaded into MGD during a broadcast. The Primary reference is always broadcast to MGD.

    14. Symbol/Name Search (Special field; see Miscellaneous for description)

Defaults

  1. Event; default is assigned
  2. Status; default is Pending
  3. Submitted by; default is current user
  4. Broadcast by; default is current user when user broadcasts symbol
  5. Broadcast Date = today when user broadcasts symbol
  6. Reference Broadcast? = Yes. The primary reference is always broadcast.

Duplicates

(Remember: Our database server is case-insensitive)
  1. Other Names duplicates are disallowed. Therefore, you cannot enter both DokR and Dokr as Other Names. Only one of these values is allowed.

Broadcasts to MGD

(Note: Refer to TR 1291 Requirements for definitions of Tier 3 and Tier 4 users and more details on each type of Broadcast.)

Tier 3 users can only broadcast symbols which they have submitted and which have a status of Pending. Tier 4 users may broadcast any symbol except a symbol with a status of Broadcast or Deleted.

For a batch broadcast, if any errors are detected during the broadcast, the entire batch fails and no symbols will be broadcast.

For the add and broadcast, if any errors are detected during the broadcast, the symbol will still be added to Nomen.

When a broadcast to MGD is successful, the Status, Broadcast by, Broadcast Date, and MGI Accession ID fields are automatically populated. No user can edit these fields manually.

  1. Add and Broadcast to MGD - add symbol to Nomen and broadcast to MGD in one step
  2. Broadcast by Symbol to MGD - broadcast one symbol to MGD
  3. Broadcast by Batch to MGD - broadcast all Approved symbols to MGD (Tier 4 only)
  4. Broadcast by Reference to MGD - broadcast all Pending (if Tier 3) or Approved (if Tier 4) symbols which contain the specified primary reference to MGD

Miscellaneous

  1. When the user adds either a Primary or Related Reference, the MGD Master Bibliography record for that Reference will be updated so that the Nomenclature data set is selected.

  2. The Symbol/Name Search field is used for querying only. When the user enters a value into this field and clicks Search, the system will take the search criteria entered and search the Symbol and Name fields for the value. Search criteria entered in other form fields are ignored when the Symbol/Name Search field is populated.

    The search fields for Symbol/Name Search can be expanded to include Names and/or Notes.

  3. To duplicate a record, select the record you want to duplicate and then de-select it. All of the data fields except Other Names, Accession Numbers, Editor's Notes and Nomenclature Coordinator Notes will remain populated. Make the changes necessary for the new record and click Add. The new record will be added.
