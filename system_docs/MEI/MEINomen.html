MGD EDITING INTERFACE: Design and Requirements - Nomenclature


Introduction

The Nomen DB is used by the MGI informatics group to enter novel and reserved mouse gene symbols and to broadcast these symbols to MGD.

Refer to TR 1291 Requirements for more information.

Data Verifications

  1. Event (see Data Verifications section on Pulldown Menus)

  2. Status (see Data Verifications section on Pulldown Menus)
    last updated: ljm 01/09/2004

    • Broadcast - Official; the symbol has been approved by Mouse Genomic Nomenclature Committee (MGNC) and/or the HUGO Gene Nomenclature Committee (HGNC) and transferred to MGD as an "official" symbol.

    • Broadcast - Interim; the symbol has not been approved by MGNC or HGNC at the time it was transferred to MGD. Symbol is temporary and awaiting further review until an official symbol is assigned. This allows data to be associated with the symbol and entered into MGI in a timely fashion.

    • In Progess; the symbol is awaiting additional work by curator or confirmation from HGNC, authors or community experts.

    • Reserved; the symbol has been approved by MGNC and is to be kept confidential/private until publication of the paper or author/MGNC/HGNC requests that it be released.

    • Approved; the symbol is a locus that was "reserved" or "in progress" and is now in MGI as either the same symbol or a different symbol. The symbol should never be broadcast to MGI since it already exists in MGI.

    • Deleted; the symbol is no longer needed for inclusion in MGI. The data associated with the symbol is needed for possible future use. Generally this is a reserved symbol, though it may have been a symbol in MGI that has been deleted in MGI.

    • Unreviewed; a BC###### symbol that was entered into Nomen via the LocusLink/MGC processing. The data associated with this symbol have not been reviewed by a curator.

  3. J Numbers (see Data Verifications section on J Numbers)

  4. Markers
    • If the Marker symbol exists in MGD or Nomen, then a warning is displayed to the user. Modifications allowed.
    • If the Marker symbol exists in MGD with the "-pending" suffix, then a warning is displayed to the user. Modifications allowed.

  5. Marker Type (see Data Verifications section on Pulldown Menus)

  6. Chromosome (see Data Verifications section on Pulldown Menus)

Defaults

  1. Event; default is assigned
  2. Status; default is In Progress
  3. Submitted by; default is current user
  4. Broadcast by; default is current user when user broadcasts symbol
  5. Broadcast Date = today when user broadcasts symbol

Duplicates

(Remember: Our database server is case-insensitive)
  1. Other Names duplicates are disallowed. Therefore, you cannot enter both DokR and Dokr as Other Names. Only one of these values is allowed.

Broadcasts to MGD

(Note: Refer to TR 1291 Requirements for definitions of Tier 3 and Tier 4 users and more details on each type of Broadcast.)

Tier 3 users can only broadcast symbols which they have submitted and which have a status of In Progress. Tier 4 users may broadcast any symbol except a symbol with a status of Broadcast, Approved or Deleted.

For the add and broadcast options, if any errors are detected during the broadcast, the symbol will still be added to Nomen.

When a broadcast to MGD is successful, the Status, Broadcast by, Broadcast Date, and MGI Accession ID fields are automatically populated. No user can edit these fields manually.

  1. Add and Broadcast Official to MGD - add symbol to Nomen and broadcast to MGD as Official in one step
  2. Add and Broadcast Interim to MGD - add symbol to Nomen and broadcast to MGD as Interim in one step
  3. Broadcast Official Symbol to MGD - broadcast one symbol to MGD and status as Official
  4. Broadcast Interim Symbol to MGD - broadcast one symbol to MGD and status as Interim

Miscellaneous

  1. When the user adds either a Primary or Related Reference, the MGD Master Bibliography record for that Reference will be updated so that the Nomenclature data set is selected.

  2. The Symbol/Name Search field is used for querying only. When the user enters a value into this field and clicks Search, the system will take the search criteria entered and search the Symbol and Name fields for the value. Search criteria entered in other form fields are ignored when the Symbol/Name Search field is populated.

    The search fields for Symbol/Name Search can be expanded to include Names and/or Notes.

  3. To duplicate a record, select the record you want to duplicate and then de-select it. The data fields will remain populated. Make the changes necessary for the new record and click Add. The new record will be added and will contain all of the information retained from the original record. Editor's Notes and Nomenclature Coordinator Notes are not duplicated.
