Introduction

MGD stores Marker information and Accession numbers for non-mouse Markers which are used in the Homology portion of the database.

From the Web Homology detail page, MGD links to species-specific databases (SSDBs) either by Marker symbol or by SSDB Accession number. Not every SSDB provides Accession numbers for their Marker objects.

SSDBs are also used to maintain MGD nomenclature and mapping information for non-mouse markers.

The Marker information and Accession numbers for the non-mouse Markers must be updated on a regular basis to reflect the SSDB changes in nomenclature, mapping position, etc.

Getting the Data Dumps

The species-specific Accession numbers are loaded into MGD by an automated process which uses data dumps provided by the SSDBs.

LocusLink

We are currently using LocusLink to update MGD human symbols, mapping and accession information. We are only loading the GDB ID from the LocusLink file.
  1. All MGD Human symbols which do not contain a GDB ID are updated with the correct GDB ID based on a symbol match between MGD and the LocusLink symbol (official or interim).

  2. All MGD Human symbols are updated from the LocusLink file based on a GDB ID match between MGD and LocusLink. The fields which are updated are:
    • symbol
    • name
    • chromosome
    • offset

  3. We output the following:
    • MGD Human Symbols updated today (LL.found.sql.rpt)
    • MGD Human Symbols w/out a match in LocusLink (LL.notfound.sql.rpt)

  4. Executed from a sybase cron on titan: 
    	/usr/local/mgi/locuslink/LLupdate.sh -l

RATMAP

We are currently using RATMAP to update MGD rat mapping and accession information. We are not updating MGD Rat symbols from RATMAP.
  1. All MGD Rat symbols which do not contain a RATMAP ID are updated with the correct RATMAP ID based on a symbol match between MGD and the RATMAP symbol.

  2. All MGD Rat symbols are updated from the RATMAP file based on a RATMAP ID match between MGD and RATMAP. The fields which are updated are:
    • chromosome
    • offset

  3. We output the following:
    • MGD Rat Symbols updated today (RATMAP.found.sql.rpt)
    • MGD Rat Symbols w/out a match in RATMAP (RATMAP.notfound.sql.rpt)
    • Conflicts between MGD Rat Symbols and RATMAP Symbols (RATMAP.conflicts.sql.rpt)
    • Potential Mouse/Rat Matches (RATMAP.potentials.sql.rpt)
    • RATMAP Symbols and GenBankIDs (RATMAP.genbankIDs.sql.rpt)

  4. In addition, we retrieve the following files from the RATMAP ftp site:
    • RATMAP Homolog file (RATMAP.Homolog.txt)

  5. Executed from a sybase cron on titan: 
    	/usr/local/mgi/ratmapload/RATMAPupdate.sh -l

GDB

We are currently using GDB to create a report of potential human homologies based on matching GDB citations with MGD citations by symbol for which no human/mouse ortholog exists in MGD.

  1. We output the following:
    • MGD Entries with Potential Human Homologies (HMD_Potentials.rpt)
