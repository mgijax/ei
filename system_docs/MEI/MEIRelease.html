MGI EDITING INTERFACE - Release Process

Purpose

This document provides procedures for preparing and installing a release of an Editing Interface Application (MGD, GXD, Nomenclature, MGI Admin).

Read MGI EDITING INTERFACE - Overview before reading this document. The overview document contains definitions and other terms which will are also used in this document.

Release Procedures

  1. Email info-mgi to inform everyone in the group of the pending release. Include the following information:
    • Ask everyone to exit the application by a certain time prior to installing the new release.
    • Ask everyone to refrain from entering the application again until you have sent additional email informing them that the release has been successfully installed.
    • Try to give an approximate time for when the new release will be available.

  2. Check the user logs. At least once a year, the logs should be backed up to tape (see Ruth) and then removed so that new logs are created. As the logs get bigger and bigger, it can be awkward to load them into an editor so that they can be read when debugging problems.

  3. Check titan to verify that there are no editing sessions running for the application being released. For example, to determine if any MGD applications are currently running: 
        titan> ps -ef | grep runmgd

    If there are applications still running and you are ready to build the release, either kill the processes (you must be root) or contact the user and ask him/her to exit the application.

  4. Build and install a Production copy of the application.

  5. Run the installed application.

  6. Once you are confident that the release has been installed successfully, email info-mgi and inform everyone in the group that the application can be used.
