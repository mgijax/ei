GXD EDITING INTERFACE: Design and Requirements - Assays

Introduction

(This information has been extracted from the GXD Schema document)

An expression assay is defined as a group of expression-related facts reported in a single reference, having a single type (e.g. Western blot), measuring the expression of a single gene and using a single probe. Within one assay, expression may be measured among different animals (having different genotypes and/or ages) and within different tissues.

If an assay is a Gel Assay (e.g. Northern), the the assay record corresponds to the gel itself.

If an assay is an In Situ (e.g. Immunohistochemistry), then the assay record corresponds to a somewhat abstract grouping of the animals and results for a single probe.

If a Gel or In Situ assay uses multiple probes, then it is split into separate assays when it is entered into the database.

Each Assay contains a nucleotide probe preparation record or an antibody preparation record. A probe (nucleotide or antibody) used in an assay is "prepared" by attaching a radioactive label to it so that it can be detected, e.g. by autoradiography.

Defaults

  1. In Situ Assays
    1. If the Strength of a Specimen Result is Absent, then the Pattern will default to Not Applicable.

  2. Genotypes
    1. If no Genotype value is entered, then the Genotype will default to Not Specified.

Deletions

There are no restrictions on deleting an Assay record.

Miscellaneous

GXD Expression Cache

Adds/Modifications

  1. Add basic Assay information (Reference, Assay Type, Marker, Notes) and Antibody/Probe Preparation and, optionally, Specimen or Gel Lane/Row information. This information must be defined before Specimen Results or Gel Band information can be entered.
  2. Any modification to an Assay record (Specimen, Specimen Results, Gel Lane, Gel Row, Gel Band, Antibody Prep, Probe Prep) will update the modification date of the master Assay record.

Probe Preps

  1. During the add of a Probe Prep record, a corresponding Probe Reference record for the J#/Probe pair will automatically be added to the Molecular Probes & Segments portion of the database.
  2. The user is no longer (as of TR 2547) required to enter the Probe Holder information, but he/she may opt to enter it using the following rules:
    • Person who gave the Probe or
    • Corresponding Author or
    • First Author

Genotypes

  1. When copying the Genotype information from one row to another, the system is actually using the same Genotype defintion for both rows. So, any modification to that Genotype definition affects all rows which have copied that Genotype column. If the user is building a new Genotype defintion for the same Strain, then the user should not perform a row copy. The user must bring up the Genotype dialog and define a new Genotype.

  2. Marker Alleles (see Data Verifications section on Marker Alleles) may be added during Genotype entry. If an entered Allele does not exist, then the Allele form is launched so that the user can immediately add the new allele. After the allele is successfully added using the Allele form, the Allele form is closed.

Anatomical Structures

  1. The editor will build the Anatomical Structure Clipboard via the Anatomical Dictionary form. Each session of the application has its own clipboard, that is each editor has his/her private clipboard.
  2. When the InSitu Results dialog or the Gel Lane table is displayed:
    1. The system will automatically load the current contents of all InSitu Result Structures or Gel Lane Structures into the side panel Anatomical Structure list. These Structures will contain the [DB] prefix.
    2. The system will automatically append the contents of the clipboard into the side panel Anatomical Structure list. These Structures will contain the [Clipboard] prefix.
    3. If a Structure exists in both the [DB] and the [Clipboard], then the Structure will only be listed once and will contain the [DB] prefix.
  3. To reload the Anatomical Structure list, click on the list label. This will perform Step 2. Reload the list if:
    1. You return to the Anatomical Dictionary form and revise the Clipboard.
    2. You return to the Anatomical Dictionary form and select another current Structure node.
    3. You return to the Anatomical Dictionary form and modify any nodes.
  4. When a particular InSitu Result or Gel Lane table row is selected:
    1. The appropriate [DB] entries are highlighted to inform the user which Structures are currently assigned to the given InSitu Result or Gel Lane.
  5. To assign a Structure to a particular InSitu Result or Gel Lane:
    1. Select the InSitu Result or Gel Lane row you wish to edit.
    2. The currently assigned Structures will be highlighted in the Structure list.
    3. Select the Structure you wish to assign to the InSitu Result or Gel Lane. A selected Structure will be highlighted.
  6. To remove a Structure from a particular InSitu Result or Gel Lane:
    1. Select the InSitu Result or Gel Lane row you wish to edit.
    2. The currently assigned Structures will be highlighted in the Structure list.
    3. De-select the Structure you wish to remove from the InSitu Result or Gel Lane. A de-selected Structure will be not be highlighted.
  7. Select Save Edits to save all modifications.
  8. Modification of Anatomical Structure assignments can be peformed simultaneously with other InSitu Result or Gel Lane modifications.

InSitu Assay

  1. Add Specimen information (Genotype, Age, Sex, Fixation, Embedding, Hybridization, Notes). Specimens and Genotypes can be added at the same time as the Assay information or subsequent to an Assay add.
  2. Add Specimen Results. Results cannot be added/modified if a Specimen does not exist.
  3. The number of Specimen Results appears in the Results column of the Specimen table.
  4. All Specimen columns except Age Range and Results can be copied from the previous row. To copy a column from the previous row, tab through the current row/column. If a value exists in that column in the previous row, then the previous row's value is copied to the current row.
  5. The InSitu Results dialog can remain displayed so the user can move back to the Anatomical Dictionary or to another Specimen row. If the user chooses another Specimen row, the InSitu Results will be refreshed to display the Results for the currently selected Specimen row. Any changes which have not been saved will be lost if the user moves to another Specimen row without clicking the Save Edits button prior to selecting the new row.
  6. All InSitu Result columns (Structures, Strength, Pattern, Image Panes, Notes) can be copied from the previous row. To copy a column from the previous row, tab through the current row/column. If a value exists in that column in the previous row, then the previous row's value is copied to the current row.

Gel Assay

  1. Add Gel Lane information (Genotype, RNA Type, Control, Sample Amount, Sex, Age, Notes). Gel Lanes cannot be added/modified if an Assay does not exist.
  2. Add Gel Row information (Units, Size, Notes). Gel Rows cannot be added/modified if an Assay does not exist.
  3. Gel Lane and Gel Row information can be added at the same time as the Assay information or subsequent to an Assay add.
  4. After the Gel Lane and Gel Row information is added, the Gel Band table will be constructed based on the number of Lanes and Rows defined.
  5. Add Gel Band information (Strength, Notes). Gel Bands cannot be added/modified if a Gel Lane does not exist. However, Gel Row and Gel Band information can be entered simultaneously.
  6. Gel Rows are automatically sorted in decreasing order based on size after the record is added or modified.
  7. All Gel Lane columns can be copied from the previous row. To copy a column from the previous row, tab through the current row/column. If a value exists in that column in the previous row, then the previous row's value is copied to the current row.
  8. All Gel Row/Band columns can be copied from the previous row's values. To copy a column from the previous row, tab through the current row/column. If a value exists in that column in the previous row, then the previous row's value is copied to the current row.
  9. When entering Gel Band Strength or Gel Band Notes, the user must specifically select the appropriate Lane column in the Gel Row table. If the interface cannot determine a Lane for the Strength or Note, then an error message will be displayed.
  10. Summary of options for adding Gel Lane, Gel Row and Gel Band information:
    1. Gel Lanes and Gel Rows can be added simultaneously
    2. Gel Rows and Gel Bands can be added simultaneously
    3. Gel Lane, Gel Rows and Gel Bands can be added simultaneously, but in this case Gel Bands can only be added for those Gel Lanes which have been defined prior to the add.

Marker Transfer

The Assay form contains a read-only version of the Marker Transfer dialog. This allows the GXD Editor to preview a potential Marker Transfer operation's effects within GXD and the rest of MGD.

Refer to the MGI Marker Transfer document for complete details on the Transfer process.
