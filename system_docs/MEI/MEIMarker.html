MGD EDITING INTERFACE: Design and Requirements - Markers


Introduction

The Markers constitute one of the backbones of the MGD database. Virtually no data is entered into the MGD database without a Marker of some kind associated with it.

Markers usually make their way into MGD via the Nomenclature process. Markers are broadcast from the Nomenclature database to MGD via a broadcast file which is processed upon user request. The broadcast file handles processing of new symbols, modified symbols, and withdrawn symbols (including splits). Withdrawals due to the discovery that one symbol is actually an allele of another symbol are usually processed directly within the Marker form, but can also be part of a broadcast file. For more information regarding the Nomenclature database and the Broadcast process, refer to:

  • Nomenclature Requirements
  • Broadcast Requirements

Fields

  • Required
    1. Marker Type; (Gene, DNA Segment, QTL, etc.)
    2. Symbol
    3. Chromosome
    4. Current Symbol
    5. Offset
    6. Name
    7. History; Nomenclature history of the Marker symbol

  • Not Required
    1. Cytogenetic Band
    2. Chromosome Committee (CC) Offset
    3. MIT Offset
    4. Notes
    5. Aliases
    6. Allele Symbol/Name
    7. Other Names
    8. References

Defaults

  1. Current Symbol; default is Symbol
  2. Offset (on add)
    • If no MGD Offset is entered and chromosome = RE, then default is -999.0
    • If no MGD Offset is entered and chromosome = UN, then default is -1.0
    • If no MGD Offset is entered and chromosome != RE, then default is -1.0
  3. History (on add); default History event is Assigned

Deletions

A Marker cannot be deleted if it is referenced by a:
  1. Experiment
  2. Molecular Probe & Segment
  3. Homology
  4. MLC entry
  5. Expression Assay

Allele Merge

The Allele Merge process allows the user to merge two pre-existing Marker Allele Symbol records into one Marker Allele Symbol record.
  1. Click on Utilities->Allele Merge to bring up the Allele Merge dialog.
  2. Enter the Marker Symbol in the Marker dialog field. This field will be verified.
  3. Enter the Old Allele Symbol in the Old Allele Symbol dialog field. The Old Allele Symbol must currently exist as an Allele Symbol for the Marker Symbol.
  4. Enter the New Allele Symbol in the New Allele Symbol dialog field. The New Allele Symbol must currently exist as an Allele Symbol for the Marker Symbol.
  5. Click Process Merge to process the merge.
  6. Click Done to return to the Marker editing form.

Modifying Chromosome Values

When modifications are made to the Chromosome value of the Marker, certain rules apply:
  1. If the Chromosome is being changed to RE, then set the Offset to -999.0.
  2. If the Chromosome is being changed to UN, then set the Offset to -999.0.
  3. If the Chromosome is being changed to W, then manage the Withdrawal dialog.
  4. If the Chromosome was RE and is now assigned, then set the Offset to -1.0.
  5. If the Chromosome was known and is changed to another known, then set the MGD and CC Offsets to -1.0. However, if the Marker is an Anchor Locus, then the Anchor record must be deleted before the Chromosome value can be modified.

Other Names/References

The Other Names/Reference table can be used to edit:
  1. Other Names which do not have References
    • Enter Other Name only. Modify record.
  2. Other Names which do have References
    • Enter Other Name and J:. Modify record.
  3. References only (additional references, MLC indexing, etc.)
    • Enter J: only. Modify record.
References which are new or are added to existing other names will become immediately visible on the Marker Reference display in the Web interface (WI). Deletions (removals) of References may not be reflected in the WI until the next day.

Miscellaneous

  1. If the current Chromosome value of a record is W, then the Chromosome value cannot be modified.

  2. Editing of the # column is implemented for the following tables:
    • Marker History

  3. Editing of UniGene accession numbers added during the electronic UniGene load is restricted. These electronically loaded accession numbers are distinguished by the reference J:57747. If the user attempts to edit the Accession numbers of a marker record from the UniGene load, the following error message will appear: 
    	Accession ID is referenced by J:57747, UniGene Database Download.
	Cannot edit.
