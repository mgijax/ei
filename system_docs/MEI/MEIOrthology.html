MGI EDITING INTERFACE: Design and Requirements - Orthology


Introduction

Orthology defines relationships of genes among different mammalian organism. This interface allows the user to define a Orthology reported in a specific Reference. Once that Orthology has been entered, the interface will attempt to merge that Orthology with other Orthologies which have at least one of the symbol/organism pairs in common. This creates a Orthology "class" thru transitivity: 
	If Mouse A is orthologous to Human A and Human A is orthologus to Rat A, 
	then Mouse A is orthologous to Rat A

Orthologies are defined by the Assay(s) which were used to determine the orthology. Therefore, one Reference may report 3 different orthologous relationships using 3 different assays. The 3 orthologies may fall under the same orthology "class", or they may not.

In this interface, it is important to remember that one selected record is equivalent to one defined orthology and not an entire orthology class.

Downloads from Organism-specific databases are used to update the Symbol attributes for non-mouse organism on a regular basis.

Defaults

The Marker table always displays the following organism:
  1. mouse, laboratory
  2. human
  3. rat
When a orthology record is displayed, all of the default organism plus all of the organism defined in the orthology are displayed. If no Symbol is associated with a default organism, then that organism is not part of the orthology record.

Default organism values cannot be overwritten by any other organism value. The row of a default organism value is reserved solely for that organism.

Searches

All fields in the editing form are searchable with the following exceptions:
  1. Numbered columns in the Marker table
  2. Numbered columns in the Assay table

Deletions

There are no restrictions on deleting a Orthology.

Modifying Symbols

Symbols for mouse can only be modified using the Marker form.

To edit symbols for non-mouse organism, use the Marker NonMouse editing form which is available from the Edit->Marker Information selection on the Orthology editing form. See the MGI EDITING INTERFACE: Design and Requirements - Non-Mouse Marker document for details about this editing form.

Adding/Modifying a Orthology

  1. Enter all of the Organism/Symbols in the Symbol table.

  2. Select all of the Assays which are used in defining the orthologies for the Symbols.

  3. Enter all of the Notes which are available.

  4. Select the Symbol/Assay combinations which define each orthology by clicking in the Assay column which corresponds to the No. column of the Symbol.

  5. A Organism/Symbol can belong to more than one Assay group, but must belong to at least one Assay group or else it will be dropped from the orthology. Therefore, to delete a Symbol from a orthology, de-select the Symbol from the Assay to which it is attached.

    If a new Assay is to be assigned to a pre-existing Orthology, insert the new Assay adjacent to the pre-existing Orthology. This will ensure that the Assay groups are assigned to the same Orthology key.

  6. After any add/modification, the system will attempt to "merge" the current orthology record with other orthologies which belong to the same "class", based on the Symbols/Organism defined.

  7. As Organism/Symbol are entered in the Symbol table, the system will detect any collisions, i.e. the entry of a Organism/Symbol which conflicts with a previously defined orthology.

    For example, if a orthology class exists for mouse, Ren1 and human, REN and the user attempts to enter mouse, Ren1 and human, REN1 and the symbol human, REN1 exists in the database, then the system will detect a conflict and report this to the user.

  8. If a Organism/Symbol needs to be replaced with a differenct symbol for a given Orthology Class, then the entire Orthology Class must be deleted and re-added.

Notes

Orthology notes are attached to unique orthology records. A unique orthology record is defined by the Organism to which the Assay is assigned.

At the present time, the user cannot attach notes to a specific orthology reference or to a specific orthology marker symbol.

Miscellaneous

  1. Setting the Display Inferred Orthologies ON before a Search will retrieve all Orthology records belonging to the same Orthology Class of the Orthology record which matches the user's search criteria.

    For example, searching for mouse, laboratory/Ren1 with the toggle OFF will retrieve 8 Orthology records where mouse, laboratory/Ren1 is a member of each Orthology record.

    Searching for mouse, laboratory/Ren1 with the toggle ON will retrieve all of the 16 Orthology records which belong to the Orthology Class of which mouse, laboratory/Ren1 is a member. mouse, laboratory/Ren1 will not necessarily be a member of every Orthology record which is retrieved, however.

  2. The Class ID for every Orthology record is displayed for the user. The user can also search for a specific Class ID. However, Class IDs are assigned by the system and cannot be modified by the user.

  3. Certain Organism's Symbols are updated from data downloads of organism-specific databases.
