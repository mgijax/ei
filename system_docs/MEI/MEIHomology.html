MGD EDITING INTERFACE: Design and Requirements - Homology


Introduction

Homology defines relationships of genes among different mammalian species. This interface allows the user to define a Homology reported in a specific Reference. Once that Homology has been entered, the interface will attempt to merge that Homology with other Homologies which have at least one of the symbol/species pairs in common. This creates a Homology "class" thru transitivity: 
	If Mouse A is homologous to Human A and Human A is homologus to Rat A, 
	then Mouse A is homologous to Rat A

Homologies are defined by the Assay(s) which were used to determine the homology. Therefore, one Reference may report 3 different homologous relationships using 3 different assays. The 3 homologies may fall under the same homology "class", or they may not.

In this interface, it is important to remember that one selected record is equivalent to one defined homology and not an entire homology class.

Downloads from Species-specific databases are used to update the Symbol attributes for non-mouse species on a regular basis (MGI Homology Loads).

Defaults

The Marker table always displays the following species:
  1. mouse, laboratory
  2. human
  3. rat
When a homology record is displayed, all of the default species plus all of the species defined in the homology are displayed. If no Symbol is associated with a default species, then that species is not part of the homology record.

Default species values cannot be overwritten by any other species value. The row of a default species value is reserved solely for that species.

Searches

All fields in the editing form are searchable with the following exceptions:
  1. Numbered columns in the Marker table
  2. Numbered columns in the Assay table

Deletions

There are no restrictions on deleting a Homology.

Modifying Symbols

Symbols for mouse can only be modified using the Marker form.

Symbols for non-mouse species can be modified/added by entering the Species/Symbol in the Symbol table and tabbing. If the Species/Symbol does not exist in the database, the user will be informed that this Species/Symbol will be added during the next modification/add of the Homology record.

If the Species/Symbol does exist, then the Chr, Cyto, Name, AccID will be displayed. The user can modify any of these values for a non-mouse species, and these changes will be be propagated to all Homology records which refer to that Species/Symbol.

There is currently no way for a user to modify a non-mouse species symbol of an existing record.

Adding/Modifying a Homology

  1. Enter all of the Species/Symbols in the Symbol table.

  2. Select all of the Assays which are used in defining the homologies for the Symbols.

  3. Enter all of the Notes which are available.

  4. Select the Symbol/Assay combinations which define each homology by clicking in the Assay column which corresponds to the No. column of the Symbol.

  5. A Species/Symbol can belong to more than one Assay group, but must belong to at least one Assay group or else it will be dropped from the homology. Therefore, to delete a Symbol from a homology, de-select the Symbol from the Assay to which it is attached.

    If a new Assay is to be assigned to a pre-existing Homology, insert the new Assay adjacent to the pre-existing Homology. This will ensure that the Assay groups are assigned to the same Homology key.

  6. After any add/modification, the system will attempt to "merge" the current homology record with other homologies which belong to the same "class", based on the Symbols/Species defined.

  7. As Species/Symbol are entered in the Symbol table, the system will detect any collisions, i.e. the entry of a Species/Symbol which conflicts with a previously defined homology.

    For example, if a homology class exists for mouse, Ren1 and human, REN and the user attempts to enter mouse, Ren1 and human, REN1 and the symbol human, REN1 exists in the database, then the system will detect a conflict and report this to the user.

    However, if the symbol human, REN1 does not exist in the database, then the symbol is added and the modification is allowed. This "back door" may need to be shut at some point in time.

  8. If a Species/Symbol needs to be replaced with a differenct symbol for a given Homology Class, then the entire Homology Class must be deleted and re-added.

Notes

Homology notes are attached to unique homology records. A unique homology record is defined by the Species to which the Assay is assigned.

At the present time, the user cannot attach notes to a specific homology reference or to a specific homology marker symbol.

Miscellaneous

  1. Setting the Display Inferred Homologies ON before a Search will retrieve all Homology records belonging to the same Homology Class of the Homology record which matches the user's search criteria.

    For example, searching for mouse, laboratory/Ren1 with the toggle OFF will retrieve 8 Homology records where mouse, laboratory/Ren1 is a member of each Homology record.

    Searching for mouse, laboratory/Ren1 with the toggle ON will retrieve all of the 16 Homology records which belong to the Homology Class of which mouse, laboratory/Ren1 is a member. mouse, laboratory/Ren1 will not necessarily be a member of every Homology record which is retrieved, however.

  2. The Class ID for every Homology record is displayed for the user. The user can also search for a specific Class ID. However, Class IDs are assigned by the system and cannot be modified by the user.

  3. Certain Species' Symbols are updated from data downloads of species-specific databases. Refer to MGI Homology Loads for a complete description of how this data is used to update Symbols and Accession numbers in MGD.
